Welcome to the **F1 Insights & Strategy Explorer** repository for CSCI 6234 (Spring 2026), Group 23. This project aims to analyze and visualize Formula 1 data using Object-Oriented Analysis and Design (OOAD) principles.

## 📁 Repository Structure

The project artifacts are structured sequentially through the four OOAD phases:

| Phase | Directory | Description |
|-------|-----------|-------------|
| **1. Requirements** | [`/Requirements`](./Requirements) | Contains Use Case diagrams and requirements descriptions (Hand-drawn and PlantUML). |
| **2. Analysis** | [`/Analysis`](./Analysis) | Features the Domain Model, 5 Robustness Diagrams, and Activity diagrams mapping out the F1 platform. |
| **3. Design** | [`/Design`](./Design) | Houses the formal implementation logic using a comprehensive Class Diagram and 5 detailed Sequence Diagrams (PlantUML + Hand-drawn styles). |
| **4. Implementation** | [`/Implementation`](./Implementation) | The working Python codebase that brings our OOAD models to life through a live, interactive F1 Streamlit dashboard. |
| **UML** | [`/UML`](./UML) | A collection of all general architecture visualizations including component and state diagrams. |

## 🚀 Live Implementation
We have translated our OOAD diagrams into a fully functional application. 

You can view the code, setup instructions, and deployment details in the **[Implementation Directory](./Implementation)**.

**Key Dashboard Features:**
- 🏁 Multi-Session Data selection
- 👥 Multi-Driver Comparative Analysis
- ⏱️ Lap Pace Analysis & Tracking
- 🔧 Visual Tyre Strategy Breakdowns

## 🛠️ Stack & Technologies
- **Modeling:** PlantUML
- **Frontend/Dashboard:** Streamlit
- **Data Source:** FastF1 Python API
- **Charts:** Plotly
- **Design Process:** Rational Unified Process (RUP) Iterations
